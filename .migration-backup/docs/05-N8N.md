# AutoBot360 — n8n Orchestration Architecture

## Overview

n8n is the **workflow orchestration layer** between Pub/Sub events and external APIs. It handles retries, branching, AI calls, and multi-step social commerce flows that would be brittle as Cloud Functions chains.

## Deployment Architecture

```
┌─────────────────────────────────────────────────────────────┐
│  n8n Cluster (GKE or Cloud Run + Memorystore Redis)         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ n8n Main    │  │ n8n Worker 1│  │ n8n Worker N│         │
│  │ (webhooks)  │  │ (executions)│  │ (executions)│         │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘         │
│         └────────────────┼────────────────┘                 │
│                          ▼                                  │
│                   Redis Queue (Bull)                          │
└──────────────────────────┬──────────────────────────────────┘
                           │
         ┌─────────────────┼─────────────────┐
         ▼                 ▼                 ▼
    Firestore          Gemini API        Social APIs
```

### Connection Model

| Integration | Method |
|-------------|--------|
| Firestore | HTTP Request node → Firestore REST API OR Code node with service account |
| Pub/Sub | Push subscription → Webhook Trigger node |
| Gemini | HTTP Request to `generativelanguage.googleapis.com` |
| Secret tokens | HTTP Request → Internal API `GET /internal/tokens/:accountId` |
| Retries | n8n Error Trigger + Failed Retry Workflow |

### User Identification

Every webhook receives:
```json
{
  "tenantId": "tenant_abc",
  "userId": "user_xyz",
  "idempotencyKey": "unique_key",
  "traceId": "trace_123"
}
```

n8n **never** stores tenant tokens in workflow static data. Always fetch via internal API at runtime.

## Workflow 1: Publish Product

### Trigger
- **Type:** Webhook (POST)
- **URL:** `https://n8n.autobot360.com/webhook/publish-product`
- **Auth:** Header `X-Webhook-Secret`

### Input JSON
```json
{
  "eventType": "publish.requested",
  "tenantId": "tenant_abc123",
  "payload": {
    "scheduledPostId": "sp_001",
    "productId": "prod_001",
    "platforms": ["instagram", "facebook"],
    "socialAccountIds": ["sa_001", "sa_002"],
    "useAiCaption": true
  },
  "idempotencyKey": "publish_sp_001_v1"
}
```

### Nodes (Sequence)

| # | Node | Type | Action |
|---|------|------|--------|
| 1 | Webhook | Webhook Trigger | Receive Pub/Sub push |
| 2 | Validate | IF | Check `tenantId` + `scheduledPostId` present |
| 3 | Idempotency | HTTP Request | `POST /internal/idempotency/check` |
| 4 | Fetch Post | HTTP Request | Firestore GET `scheduled_posts/{id}` |
| 5 | Update Status | HTTP Request | PATCH status → `processing` |
| 6 | Fetch Product | HTTP Request | Firestore GET `products/{productId}` |
| 7 | AI Caption | Execute Workflow | → AI Caption Workflow |
| 8 | Loop Platforms | Split In Batches | For each platform |
| 9 | Social Publish | Execute Workflow | → Social Media Publish Workflow |
| 10 | Save Social Post | HTTP Request | Firestore POST `social_posts` |
| 11 | Update Scheduled | HTTP Request | PATCH status → `published` |
| 12 | Publish Event | HTTP Request | Pub/Sub `publish.completed` |
| 13 | Start Monitoring | HTTP Request | Trigger Comment Monitoring registration |
| 14 | Error Handler | Error Trigger | → Failed Retry Workflow |

### APIs Used
- Firestore REST / Internal API
- Internal token API
- Pub/Sub publish (via internal API)

### Firestore Collections
- `scheduled_posts`, `social_posts`, `products`, `idempotency_keys`

### Retry Strategy
- Max 3 attempts, exponential backoff: 30s, 120s, 600s
- On final failure: status → `failed`, publish `publish.failed`

### Output JSON
```json
{
  "success": true,
  "tenantId": "tenant_abc123",
  "scheduledPostId": "sp_001",
  "socialPosts": [
    {
      "platform": "instagram",
      "platformPostId": "ig_178901234",
      "permalink": "https://instagram.com/p/..."
    }
  ],
  "executionId": "exec_789"
}
```

## Workflow 2: AI Caption

### Trigger
- **Type:** Execute Workflow Trigger (called by Publish Product)

### Input JSON
```json
{
  "tenantId": "tenant_abc123",
  "product": {
    "title": "Handcrafted Leather Wallet",
    "description": "Premium full-grain leather...",
    "price": 2499,
    "tags": ["leather", "wallet", "handmade"]
  },
  "platform": "instagram",
  "tone": "engaging"
}
```

### Nodes

| # | Node | Action |
|---|------|--------|
| 1 | Execute Trigger | Receive product data |
| 2 | Build Prompt | Code | Construct Gemini prompt |
| 3 | Gemini API | HTTP Request | `POST /v1beta/models/gemini-2.0-flash:generateContent` |
| 4 | Parse Response | Code | Extract caption + hashtags |
| 5 | Validate Length | IF | Instagram: max 2200 chars |
| 6 | Return | Set | Output caption object |

### Output JSON
```json
{
  "caption": "✨ Elevate your everyday carry...",
  "hashtags": ["#leatherwallet", "#handmade", "#luxury"],
  "fullText": "✨ Elevate your everyday carry...\n\n#leatherwallet #handmade"
}
```

## Workflow 3: Social Media Publish

### Trigger
- Execute Workflow (per platform)

### Input JSON
```json
{
  "tenantId": "tenant_abc123",
  "platform": "instagram",
  "socialAccountId": "sa_001",
  "caption": "...",
  "mediaUrls": ["https://storage.../image1.jpg"],
  "productId": "prod_001"
}
```

### Nodes

| # | Node | Action |
|---|------|--------|
| 1 | Get Token | HTTP | `GET /internal/tokens/{socialAccountId}` |
| 2 | Platform Switch | Switch | Route by platform |
| 3a | IG Container | HTTP | Meta API create media container |
| 3b | IG Publish | HTTP | Meta API publish container |
| 4 | Wait | Wait 5s | Media processing |
| 5 | Get Permalink | HTTP | Fetch post permalink |
| 6 | Return | Set | platformPostId, permalink |

### Retry
- 429 rate limit: wait `Retry-After` header
- 401: trigger Token Refresh Workflow, retry once

| 8 | Analytics | HTTP | Increment comment counter |

## Workflow 5: AI Reply

### Nodes
1. Fetch product context + store policies
2. Build conversation prompt with Gemini
3. IF auto-reply enabled → Post reply via platform API
4. ELSE → Save to `ai_replies` with `approved: false`
5. Save `ai_replies` document
6. Notify seller if manual approval needed

## Workflow 8: Razorpay Payment

### Trigger
- Pub/Sub `payment.success` OR Razorpay webhook forwarded

### Nodes
1. Verify Razorpay signature (Code node HMAC)
2. Idempotency check on `razorpay_payment_id`
3. Update `payments` status → `captured`

## Workflow 12: Token Refresh

### Trigger
- Cloud Scheduler daily → Pub/Sub `token.expiring`

### Nodes
1. Query accounts where `tokenExpiresAt < now + 7 days`
2. Per account: refresh OAuth token via platform API
3. Update Secret Manager secret version
4. Update Firestore `social_accounts.tokenExpiresAt`
5. On failure: set status `error`, notify user

---

## Workflow 13: Analytics Sync

### Trigger
- Hourly scheduler

### Nodes
1. For each tenant (batched): fetch platform insights APIs
2. Aggregate engagement metrics
3. Upsert `analytics/{tenantId}/daily/{date}`
4. Update `social_posts.engagement`

---

## Workflow 14: Failed Retry

### Trigger
- Error Trigger workflow (global)
- Pub/Sub `publish.failed`

### Nodes
1. Parse error context
2. IF retryCount < 5: increment, exponential backoff wait, re-trigger original
3. ELSE: move to DLQ, create notification, PagerDuty alert

### Backoff Schedule
`30s → 2m → 10m → 1h → 6h`

---

## Workflow 15: Scheduled Post

### Trigger
- Cron: every minute (n8n Schedule Trigger) OR Pub/Sub from Cloud Scheduler

### Nodes
1. HTTP: `GET /internal/scheduled-posts/due?limit=100`
2. Split In Batches
3. For each: publish to `publish.requested` (or directly execute Publish Product)
4. Log batch execution metrics

## Webhook Security

```
Headers required:
  X-Webhook-Secret: {from Secret Manager}
  X-Tenant-Id: {tenantId}
  X-Idempotency-Key: {unique}
  X-Cloud-Trace-Context: {trace}

Validation (first node):
  IF secret !== $env.WEBHOOK_SECRET → Stop, return 401
  IF tenantId missing → Stop, return 400
```

## Monitoring n8n

| Metric | Source | Alert |
|--------|--------|-------|
| Execution success rate | n8n API / Prometheus | < 95% |
| Queue depth | Redis INFO | > 5000 |
| Avg execution time | n8n metrics | > 60s |
| Failed executions | Error Trigger → Firestore | > 10/hour |

**Execution logging:** Every workflow writes to `workflow_executions/{id}` with status, duration, input/output hashes (not full PII).

## Pub/Sub → n8n Bridge (Cloud Function)

```typescript
// functions/pubsub-to-n8n/index.ts
export async function handleMessage(message: Message) {
  const data = JSON.parse(Buffer.from(message.data, 'base64').toString());
  const workflowMap: Record<string, string> = {
    'publish.requested': '/webhook/publish-product',
    'comment.received': '/webhook/comment-monitoring',
    'payment.success': '/webhook/razorpay-payment',
    'order.created': '/webhook/order-creation',
    'token.expiring': '/webhook/token-refresh',
    'analytics.sync': '/webhook/analytics-sync',
  };
  const path = workflowMap[data.eventType];
  await fetch(`${N8N_BASE_URL}${path}`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Webhook-Secret': process.env.N8N_WEBHOOK_SECRET!,
      'X-Tenant-Id': data.tenantId,
      'X-Idempotency-Key': data.idempotencyKey,
    },
    body: JSON.stringify(data),
  });
  message.ack();
}
```
